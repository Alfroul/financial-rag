### 阶段 4：功能 Slice — 评测与优化

> **独立会话指令**：`阅读 CONTEXT.md 和 docs/stages/agent/stage-4.md，完成所有任务后确认完成`

**目标**：构建 Agent 评测数据集，量化 Agent 模式在多步推理任务上的表现，调优 Prompt 稳定性。

**前置依赖**：阶段 3 已完成（ReAct 循环可用）。

**任务清单**：

- [x] 4.1 构建评测数据集 `data/eval/agent_eval.jsonl`
  - 设计 15-20 条多步推理任务，覆盖：
    - **对比分析**（3-5条）："对比茅台和五粮液的ROE趋势，哪个更值得投资"
    - **因果推理**（3-5条）："2024年白酒行业整体下滑的原因是什么"
    - **指标计算**（3-5条）："茅台的净利润率是多少，和行业平均比如何"
    - **综合分析**（3-5条）："帮我分析买入茅台的风险和收益"
  - 每条数据格式：
    ```json
    {
      "task": "任务描述",
      "expected_tools": ["financial_search", "knowledge_graph", "calculator"],
      "expected_steps": 3,
      "evaluation_criteria": ["包含两家公司ROE数据", "有对比计算", "给出投资建议"]
    }
    ```

- [x] 4.2 编写 Agent 评测脚本 `scripts/agent_benchmark.py`
  - 加载评测数据集
  - 对每条任务：
    - 运行 agent_query()
    - 记录：最终答案、使用的工具序列、步数、耗时
    - LLM-as-judge 评估答案质量（1-5分）
  - 生成评测报告：
    - 工具使用分布
    - 平均步数 / 超步数比例
    - 答案质量评分
    - 典型成功/失败案例

- [x] 4.3 Prompt 调优
  - 根据评测结果调整：
    - System prompt 的工具描述（LLM 是否正确选择工具）
    - Few-shot 示例（是否需要更多/更少的示例）
    - Action 格式说明（Qwen3-8B 的格式遵循度）
  - 调优目标：
    - Action 解析成功率 > 90%
    - 工具选择准确率 > 85%
    - 无效重复步数 < 10%

- [x] 4.4 边界情况处理
  - 空任务 → 直接返回提示
  - 超长任务（> 500字） → 截断或拒绝
  - 纯闲聊（"你好"） → 不调工具直接回答
  - 纯知识问答（"GDP是什么"） → 引导走问答模式而非 Agent

- [x] 4.5 验证
  - 评测数据集完整（≥ 15 条）
  - 评测脚本可运行，生成报告
  - Prompt 调优后 Action 解析成功率达标
  - 边界情况处理完整

**验收标准**：
- 评测数据集 ≥ 15 条多步推理任务
- 评测脚本可运行并生成结构化报告
- Action 解析成功率 > 90%
- 边界情况均有处理
